Payroll Assistant Architecture

Current implementation: authenticated dashboard, payslip extraction, deterministic finance logic, source-aware narration, audit logging, and configurable LLM providers.

React Dashboard

Login, payslip upload, salary summary, chat, month comparison, 80C simulator, proof checklist.

Auth + Ownership

Fake bearer token, EMPLOYEE/PAYROLL_ADMIN roles, employeeId ownership checks.

Payslip Upload API

Protected upload endpoint with rate limit, file size limit, and audit metadata.

Text Extraction

PDF text layer, DOCX/TXT parsing, image/scanned PDF OCR with optional warmup.

PII Sanitizer

Masks common PII before OpenAI text extraction. Audit log avoids raw text/base64.

LLM Provider Switch

USE_OPENAI=true keeps OpenAI flow. USE_OPENAI=false calls company wrapper.

Extraction LLM

Returns structured payroll JSON. Wrapper can receive raw PDF/image base64.

SQLite JSON Store

Stores extracted payroll payload with employeeId metadata for ownership.

Finance Logic

Salary summary, validation warnings, total earnings/deductions, calculated net.

Business Modules

Month comparison, Section 80C simulator, investment proof checklist, history cache.

Dashboard APIs

Payroll summary, history, comparison, tax simulation, proof checklist responses.

User Query

Chat question with recent context and authenticated employee scope.

Source-Aware LLM Narrator

Prompt includes payroll_json + calculation_json + source_reference_json. If LLM fails, deterministic fallback answers from saved payroll data.

Answer For User

Natural-language answer with cited payslip/calculation sources when available.

Security

CORS allowlist, rate limits, question validation, RBAC ownership.

Audit

Login, upload, summary, chat, tax, and checklist actions logged without raw PII.

Deterministic

Finance calculations, tax logic, proof checklist, and comparisons stay outside the LLM.

LLM Use

Only extraction and narration use LLM providers.
